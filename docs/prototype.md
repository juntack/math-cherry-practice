# 最小プロトタイプ仕様

## 目的

最初のプロトタイプでは、アプリ全体ではなく「1問を最後まで解ける画面」に絞る。

確認したいこと:

- 学習ロジックとUIを分離できるか
- 数字ボタンだけで操作が成立するか
- さくらんぼ図が理解しやすいか
- 正誤フィードバックとヒントが学習の妨げにならないか

## 最初の対象

- 足し算のみ
- 固定問題 `8 + 7`
- 段階3: 途中計算
- 数字ボタン入力
- さくらんぼ図表示
- ヒント表示
- 正解時に○印と「ピンポーン」音
- 不正解時に×印と「ブブーッ」音

## 現在の実装状況

最初の固定問題プロトタイプを作成した後、MVPに向けて以下を追加した。

- 足し算、引き算、混合の出題モード選択
- 3問、5問、10問の問題数選択
- 段階1「かずをわける」、段階2「10をつくる」、段階3「じゅんばん」の選択
- 繰り上がり足し算のランダム問題生成
- 繰り下がり引き算のランダム問題生成
- 引き算は「十何を10と一の位に分ける方式」
- 複数問題を順番に解くセッション進行
- セッション終了画面
- 段階別ステップ生成、足し算・引き算ロジックの単体テスト
- SVGの枝を使った、教材図に近いさくらんぼ図表示

## 画面フロー

1. `8 + 7` を表示する
2. 「8を10にするには、あといくつ？」と問う
3. `2` を選ぶ
4. `7` を `2` と `□` に分ける表示にする
5. 「7を2といくつにわける？」と問う
6. `5` を選ぶ
7. `8 + 2 = 10` を表示する
8. `10 + 5 = □` を問う
9. `15` を選ぶ
10. 正解フィードバックを表示する
11. 「もういちど」「つぎのもんだい」ボタンを表示する

## 不正解時

例: 最初に `3` を選んだ場合

1回目:

```txt
8を10にするには、あといくつかな？
```

2回目以降:

```txt
8を10にするには、あと2だよ。
```

ただし、回答欄に正解を自動表示しない。

## まだ含めないもの

- ドラッグ操作
- 10フレーム
- 成績保存
- 補助表示を段階的に減らすモード
- さくらんぼ計算図の細かな見た目調整

## 完了条件

- 固定問題 `8 + 7` を3ステップで最後まで解ける
- 正解、不正解、ヒントが動作する
- UIとロジックが分離されている
- `needed = 2`、`remainder = 5`、`answer = 15` がロジック側で生成されている
- 主要ロジックに単体テストがある
