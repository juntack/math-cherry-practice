# 最小プロトタイプ仕様

## 目的

最初のプロトタイプでは、アプリ全体ではなく「1問を最後まで解ける画面」に絞る。

確認したいこと:

- 学習ロジックとUIを分離できるか
- 数字ボタンだけで操作が成立するか
- さくらんぼ図が理解しやすいか
- 正誤フィードバックとヒントが学習の妨げにならないか

## 対象

- 足し算のみ
- 固定問題 `8 + 7`
- 段階3: 途中計算
- 数字ボタン入力
- さくらんぼ図表示
- ヒント表示
- 正解時に○印と「ピンポーン」音
- 不正解時に×印と「ブブーッ」音

## 画面フロー

1. `8 + 7` を表示する
2. 「8を10にするには、あといくつ？」と問う
3. `2` を選ぶ
4. `7` を `2` と `□` に分ける表示にする
5. 「7を2といくつにわける？」と問う
6. `5` を選ぶ
7. `8 + 2 = 10` を表示する
8. `10 + 5 = □` を問う
9. `15` を選ぶ
10. 正解フィードバックを表示する
11. 「もういちど」「つぎのもんだい」ボタンを表示する

## 不正解時

例: 最初に `3` を選んだ場合

1回目:

```txt
8を10にするには、あといくつかな？
```

2回目以降:

```txt
8を10にするには、あと2だよ。
```

ただし、回答欄に正解を自動表示しない。

## プロトタイプで含めないもの

- ランダム問題
- 引き算
- 混合モード
- 複数画面構成
- セッション管理
- ドラッグ操作
- 10フレーム
- 成績保存

## 完了条件

- 固定問題 `8 + 7` を3ステップで最後まで解ける
- 正解、不正解、ヒントが動作する
- UIとロジックが分離されている
- `needed = 2`、`remainder = 5`、`answer = 15` がロジック側で生成されている
- 主要ロジックに単体テストがある
