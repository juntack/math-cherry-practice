# 最小プロトタイプ仕様

## 目的

最初のプロトタイプでは、アプリ全体ではなく「1問を最後まで解ける画面」に絞る。

確認したいこと:

- 学習ロジックとUIを分離できるか
- テンキー型の数字ボタンだけで操作が成立するか
- さくらんぼ図が理解しやすいか
- 正誤フィードバックとヒントが学習の妨げにならないか

## 最初の対象

- 足し算のみ
- 固定問題 `8 + 7`
- 段階3: 途中計算
- 暗算モードでは式だけを表示して答えを問う
- テンキー型の数字ボタン入力
- さくらんぼ図表示
- ヒント表示
- 正解時に○印と「ピンポーン」音
- 不正解時に×印と「ブブーッ」音
- 1問の全ステップが終わった後の完了表示

## 現在の実装状況

最初の固定問題プロトタイプを作成した後、MVPに向けて以下を追加した。

- 足し算、引き算、混合の出題モード選択
- 3問、5問、10問の問題数選択
- 補助量「おおい」「ふつう」「すくない」の選択
- やりかた「さくらんぼ」「あんざん」の選択
- 段階1/2/3のステップ生成ロジック。ただし画面上の段階選択は一旦非表示
- 繰り上がり足し算のランダム問題生成
- 繰り下がり引き算のランダム問題生成
- 足し算の答えは19まで扱う
- 引き算の分解対象は18まで扱う
- 引き算は「十何を10と一の位に分ける方式」
- 複数問題を順番に解くセッション進行
- セッション終了画面
- 段階別ステップ生成、足し算・引き算ロジックの単体テスト
- SVGの枝を使った、教材図に近いさくらんぼ図表示
- 5マス×2段の10フレーム最小表示
- 現在の問いに合わせて、主表示を10フレーム・さくらんぼ図・途中式に切り替える
- 補助がふつうの場合は、10フレームを使う問いで小さい10フレームを残す
- 補助が少ない場合は、図を抑えて式中心の主表示にする
- 児童向けの主要UI文言をひらがな中心に調整
- 現在の問題内で何ステップ目か分かる点の進行表示
- 練習途中でトップへ戻るボタン
- 暗算モードでは、式だけを表示して答えを問う
- 暗算モードでは、ヒントのりょうを無効表示にする
- 練習中にセッション内の正解数を表示する
- タブレット・PCでは、練習画面を問題表示と回答欄の2カラムにして、スクロールせず答えやすくする

## 画面フロー

1. `8 + 7` を表示する
2. 「8を10にするには、あといくつ？」と問う
3. `2` を選ぶ
4. `7` を `2` と `□` に分ける表示にする
5. 「7を2といくつにわける？」と問う
6. `5` を選ぶ
7. `8 + 2 = 10` を表示する
8. `10 + 5 = □` を問う
9. `15` を選ぶ
10. 正解フィードバックを表示する
11. 1問がせいかいしたことを表示する
12. 「つぎへ」または「おしまい」ボタンを表示する

## 不正解時

例: 最初に `3` を選んだ場合

1回目:

```txt
8を10にするには、あといくつかな？
```

2回目以降:

```txt
8を10にするには、あと2だよ。
```

ただし、回答欄に正解を自動表示しない。

## まだ含めないもの

- ドラッグ操作
- 成績保存
- 補助表示をさらに段階的に減らすモード
- 10フレームやさくらんぼ図の細かなアニメーション

## 完了条件

- 固定問題 `8 + 7` を3ステップで最後まで解ける
- 正解、不正解、ヒントが動作する
- UIとロジックが分離されている
- `needed = 2`、`remainder = 5`、`answer = 15` がロジック側で生成されている
- 主要ロジックに単体テストがある
