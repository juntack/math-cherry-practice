# 決定事項・未確定事項

## 決定事項

- 技術構成は Vite + React + TypeScript + Vitest
- タブレット優先で設計する
- PCブラウザにも対応する
- 初期版の入力は数字ボタンのみ
- 将来的にドラッグ操作を検討する
- 問題数は可変設定にする
- MVPでは `3問 / 5問 / 10問` の選択式を基本にする
- 引き算の初期方式は「十何を10と一の位に分ける方式」
- 10フレームはMVPでは必須にしない
- ヒントは段階的に具体化するが、答えは表示しない
- 正解時は「ピンポーン」音と○印
- 不正解時は「ブブーッ」音と×印
- 出題モードは足し算、引き算、混合をラジオボタンで選択する
- GitHubリポジトリは適切なタイミングでユーザーに用意を依頼する

## 未確定事項

- アプリ名の正式名称
- UIの色、フォント、細かな見た目
- 効果音をWeb Audio APIで生成するか、音声ファイルにするか
- 1セッションのデフォルト問題数
- 足し算と引き算の難易度順序
- Fire HD向けAPK化をいつ行うか
- GitHub上のリポジトリ名

## GitHubリポジトリ依頼タイミング

以下のいずれかの段階で、ユーザーにGitHubリポジトリ作成を依頼する。

- 設計ドキュメントが固まった後
- 最小プロトタイプ実装前
- 最小プロトタイプが動作した後

推奨は、最小プロトタイプ実装前にGitHubリポジトリを用意すること。
