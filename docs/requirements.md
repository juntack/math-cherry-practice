# 要件定義

## 目的

小学1年生が、繰り上がりのある足し算と繰り下がりのある引き算を、さくらんぼ計算を通じて視覚的に理解する。

主目的は答えを速く出すことではなく、数を分解し、10のまとまりを作り、残りと合わせて答えを求める考え方を身につけることである。

## 対象ユーザー

- 小学1年生
- 繰り上がり足し算の学習初期の児童
- 繰り下がり引き算の学習初期の児童
- タブレットまたはPCで利用する児童
- 必要に応じて保護者が横で補助する

## 対象問題

### 繰り上がりの足し算

形式は `a + b`。

条件:

- `1 <= a <= 9`
- `1 <= b <= 9`
- `a + b >= 10`

考え方:

- `needed = 10 - a`
- `remainder = b - needed`
- `a + needed = 10`
- `10 + remainder = answer`

例: `8 + 7`

- `8` を10にするにはあと `2`
- `7` を `2` と `5` に分ける
- `8 + 2 = 10`
- `10 + 5 = 15`

### 繰り下がりの引き算

形式は `a - b`。

条件:

- `11 <= a <= 18`
- `1 <= b <= 9`
- `a` の一の位 `< b`

初期方式:

- `a` を `10` と一の位に分ける

考え方:

- `ones = a - 10`
- `remaining = 10 - b`
- `answer = remaining + ones`

例: `13 - 8`

- `13` を `10` と `3` に分ける
- `10 - 8 = 2`
- `2 + 3 = 5`

将来的には、引く数を分ける方式も追加できる構造にする。

## 操作要件

- 文字入力は原則使用しない
- 初期版は数字ボタンのみで回答する
- 数字ボタンは `0` から `20` 程度までを必要に応じて表示する
- タッチしやすい大きさにする
- 将来的に数字カードのドラッグ操作を検討できる設計にする

## 正誤フィードバック

- 正解時は「ピンポーン」と鳴らし、○印を表示する
- 不正解時は「ブブーッ」と鳴らし、×印を表示する
- 1問の全ステップが終わったら、問題単位でせいかいしたことを表示する
- 不正解時に答えそのものは表示しない
- 誤答内容に応じてヒントを出す

## ヒント要件

ヒントは段階的に具体化するが、3回目以降も答えは表示しない。

例: `8 + 7` で `7` を `3` と `4` に分けた場合

1回目:

```txt
8を10にするには、あといくつかな？
```

2回目以降:

```txt
8を10にするには、あと2だよ。
7を2といくつに分ければよいかな？
```

## 誤答分類

少なくとも以下を区別する。

- 10の補数を間違えた
- 分解した2数の和が元の数と一致しない
- 10を作った後の残りを間違えた
- 最後の計算を間違えた
- 足し算と引き算の操作を混同した

## 表示要件

- さくらんぼ図を表示する
- 10フレームを表示できる構造にする
- 10フレームは5マス x 2段の枠で表現する
- 現在の問いに合わせて、主表示を切り替える
  - 10を作るために「あといくつ」を考える場面では10フレームを表示する
  - 数を2つに分ける場面ではさくらんぼ図を表示する
  - ヒントのりょうが「ふつう」の場合は、10フレームを使う問いで小さい10フレームと式を並べる
  - 補助が少ない設定では、図を抑えて式中心の表示にする
- 現在の問題内で何ステップ目か、点の進行表示で分かるようにする

## 設定要件

開始前または設定画面で以下を選べるようにする。

- 出題モード: たしざん、ひきざん、どちらも
- 問題数: 3もん、5もん、10もん
- ヒントのりょう: おおい、ふつう、すくない

画面上の児童向け文言は、小学1年生が読みやすいように、できるだけひらがなを使う。

## 学習段階の扱い

教材仕様としては以下の段階を持つ。

- 段階1: 数の分解
- 段階2: 10を作る分解
- 段階3: 途中計算

現状の画面では段階選択を表示せず、段階3「途中計算」を固定で使用する。段階1と段階2は、将来戻せるようにステップ生成ロジックとテストを残す。
